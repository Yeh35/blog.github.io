title: gradle 마법해제하기-Build core

이어서..

전편에서 gradle의 아키텍쳐 설계와 gradle Lifecycle(생명주기)에 대해서 다뤘고 이번 편에서는 Task에 대해서 자세하게 다뤄보려고 한다.

Task?

앞에서 task는 Gradle의 실행단위라고 하였다. 실행단위를 어떤 일을 하는데 하나의 작업 목록이라고 생각하면 편하다.

Gradle을 실행되면 초기화 > 구성 > 실행 단계로 이어진다고 배웠다. 그리고 실행단계에서 실질적으로 우리가 원하는 일(행동)을 하는 것은 task다. (사실 모든건 task를 실행하기 위한 준비일 뿐이였다..)

처음에 Console에서 gradle init 명령을 통해 프로젝트를 생성 했었을 것이다. 사실 init도 하나의 Task이다. init(생성) 작업을 해주는 task

Console에 gradle [task] 명령을 통해 원하는 task를 실행할 수 있다. 기본 제공하는 task 몇 개 더 소개드리겠다. (해보세용~)

  • gradle tasks : 내장 task 리스트를 보여준다.
  • gradle help –task (task명) : task명으로 작성한 task의 도움말 보여준다.
  • gradle build : 빌드한다.
  • gradle clean : 빌드 디렉토리를 삭제한다

예제 다시보기

아래의 예제를 이전글에서 봤을 것이다.

// build.gradle file
print "Hello\n"

task testTask {
    print "testTask running\n"

    doLast {
        print "Task do Last\n"
    }
    
    doFirst {
        print "Task do First\n"
    }
}

print "finish Configure\n"

..\project> gradle testTask

> Configure project :
Hello
testTask running
finish Configure

> Task :testTask
Task do First
Task do Last

BUILD SUCCESSFUL in 1s
1 actionable task: 1 executed

print "Task do Last\n"는 구성단계에서 실행이 되고 print "Task do Last\n"와 print "Task do First\n"는 실행단계에서 실행이 되었다.

이유를 알기 위해서는 task의 특성을 알아야한다.

  • task는 TaskContainer

실행단계에서 실행된 이유는 doLast와 doFirst라는 공식문서 Task Methods를 보면 더 다양한 메소드를 볼 수 있다.

Task의 특성과

Task 그래프

어떤 작업을 할때 이 작업을 하기위해서 선행되야 하는 작업이 있을 수 있다
그럼 그런 선행되는 작업을 묶고 내가 원하는 작업을 하기전 호출 시켜주는 것이다.
Gradle에서 Task들간에 관계는 DAG(Directed acyclic graph)로 작동한다

    // build.gradle
    task taskX {
        dependsOn 'taskY'
        doLast {
            println 'taskX'
        }
    }
    task taskY {
        doLast {
            println 'taskY'
        }
    }

    // 출력
    > gradle taskX 
    taskY 
    taskX
